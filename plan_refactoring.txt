Создать папки app/routers, app/services, app/keyboards, app/core, app/db и перенести туда соответствующие файлы (например, questions.py → app/data/questions.py, db.py → app/db/users_repo.py).

В каждом файле из app/routers/*.py создать router = Router(name=__name__) и перенести туда хендлеры по смыслу (например, start_handler → routers/start.py, menu_handler → routers/menu.py).​

В app/main.py оставить только инициализацию Bot, Dispatcher, подключение роутеров через dp.include_router(...) и dp.start_polling(...).​

Вынести генерацию клавиатур в app/keyboards/*, чтобы хендлеры не собирали разметку кнопок вручную (меню/ответы/“прочитать полностью”).

Вынести логику теста в app/services/test_service.py: функции “уровень по score”, “картинка по score”, “preview 3–4 строки + …”, “полный текст”, чтобы в хендлерах остались только вызовы сервисов и отправка сообщений.

Для “уведомлять админа только если пользователь новый” сделать user_exists(telegram_id) в users_repo.py и вызывать проверку из notify_service.py перед отправкой сообщения.

(Опционально, но правильно) заменить SESSIONS на FSM storage/персистентное хранение, потому что память теряется при перезапуске; aiogram поддерживает разные хранилища для FSM.​

После каждого шага: запуск бота и быстрый прогон “/start → меню → тест → результат”, чтобы ловить ошибки импорта/регистрации роутеров сразу.
